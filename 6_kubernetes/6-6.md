# WordPressをKubernetes上に構築してブラウザから参照できるようにしてください
# ただし、WordPressのアプリケーションが落ちてもデータが永続化されるようにKubernetes上にMySQLを構築し、それと接続するようにしてください

### ■OCI上でMountTargetとFSS Exportを作成(省略)

OCIでMountTargetとFSS Exportを作成して、MountTargetのOCIDとIPアドレスを控えておく

### ■WordPress、MySQLマニフェストを作成

#### Wordpress Namespaceマニフェスト

[wordpress_namespace.yaml](./manifest_wordpress/wordpress_namespace.yaml)

```yaml
apiVersion: v1
kind: Namespace
metadata:
  creationTimestamp: null
  name: wordpress
spec: {}
```

#### MySQL用Deploymentマニフェスト

[mysql_deployment.yaml](./manifest_wordpress/mysql_deployment.yaml)

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: wordpress-mysql-deployment
  namespace: wordpress
  labels:
    app: wordpress
spec:
  replicas: 1
  selector:
    matchLabels:
      app: wordpress
      tier: mysql
  strategy:
    type: Recreate
  template:
    metadata:
      labels:
        app: wordpress
        tier: mysql
    spec:
      containers:
      - image: mysql:8.0
        name: mysql
        resources:
          requests:
            memory: "768Mi"
            cpu: "500m"
        env:
        - name: MYSQL_ROOT_PASSWORD
          valueFrom:
            secretKeyRef:
              name: mysql-pass
              key: password
        - name: MYSQL_DATABASE
          value: wordpress
        - name: MYSQL_USER
          value: wordpress
        - name: MYSQL_PASSWORD
          valueFrom:
            secretKeyRef:
              name: mysql-pass
              key: password
        ports:
        - containerPort: 3306
          name: mysql
        volumeMounts:
        - name: mysql-persistent-storage  # Block Storage
          mountPath: /var/lib/mysql
      volumes:
      - name: mysql-persistent-storage  # Block storage
        persistentVolumeClaim:
          claimName: mysql-pv-claim
```

#### MySQL用PVCマニフェスト

[mysql_pvc.yaml](./manifest_wordpress/mysql_pvc.yaml)

```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: mysql-pv-claim
  namespace: wordpress
  labels:
    app: wordpress
spec:
  storageClassName: "oci-bv"
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 70Gi
```

#### MySQL用Serviceマニフェスト

[mysql_service.yaml](./manifest_wordpress/mysql_service.yaml)

```yaml
apiVersion: v1
kind: Service
metadata:
  name: wordpress-mysql-service
  namespace: wordpress
  labels:
    app: wordpress
spec:
  ports:
    - port: 3306
      targetPort: 3306
  selector:
    app: wordpress
    tier: mysql
  clusterIP: None
```

#### Workdpress用Deploymentマニフェスト

[wordpress_deployment.yaml](./manifest_wordpress/wordpress_deployment.yaml)

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: wordpress
  namespace: wordpress
  labels:
    app: wordpress-deployment
spec:
  replicas: 2
  selector:
    matchLabels:
      app: wordpress-app
      tier: frontend
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxUnavailable: 1
  template:
    metadata:
      labels:
        app: wordpress-app
        tier: frontend
    spec:
      containers:
      - image: wordpress:6.4.2-apache
        name: wordpress
        resources:
          requests:
            memory: "256Mi"
            cpu: "400m"
        env:
        - name: WORDPRESS_DB_HOST
          value: wordpress-mysql-service
        - name: WORDPRESS_DB_PASSWORD
          valueFrom:
            secretKeyRef:
              name: mysql-pass
              key: password
        - name: WORDPRESS_DB_USER
          value: wordpress
        ports:
        - containerPort: 80
          name: wordpress
        volumeMounts:
        - name: wordpress-persistent-storage-fss  # FSS
          mountPath: /var/www/html
      volumes:
      - name: wordpress-persistent-storage-fss  # FSS
        persistentVolumeClaim:
          claimName: oke-fsspvc
```

#### Wordpress用PVマニフェスト

[wordpress_fss_pv.yaml](./manifest_wordpress/wordpress_fss_pv.yaml)

```yaml
apiVersion: v1
kind: PersistentVolume
metadata:
 name: oke-fsspv
 namespace: wordpress
spec:
 storageClassName: oci-fss
 capacity:
  storage: 10Mi
 accessModes:
  - ReadWriteMany
 mountOptions:
  - nosuid
 nfs:
# Replace this with the IP of your FSS file system in OCI
  server: 10.0.10.220
# Replace this with the Path of your FSS file system in OCI
  path: "/wordpress-fss"
  readOnly: false
```

```yaml
 name: oke-fsspvc
 namespace: wordpress

 storageClassName: oci-fss
 accessModes:
 - ReadWriteMany
 resources:
  requests:
 # Although storage is provided here it is not used for FSS file systems

```

#### Wordpress用Service(Load Balancer)マニフェスト

[wordpress_lb.yaml](./manifest_wordpress/wordpress_lb.yaml)

```yaml
apiVersion: v1
kind: Service
metadata:
  creationTimestamp: null
  name: wordpress-lb
  namespace: wordpress
  labels:
    app: wordpress
  annotations:
    oci.oraclecloud.com/load-balancer-type: "lb"
    service.beta.kubernetes.io/oci-load-balancer-internal: "false" # "true"
    service.beta.kubernetes.io/oci-load-balancer-shape: "flexible"
    service.beta.kubernetes.io/oci-load-balancer-shape-flex-min: "10"
    service.beta.kubernetes.io/oci-load-balancer-shape-flex-max: "30"
spec:
  ports:
  - port: 30097
    protocol: TCP
    targetPort: 80
  selector:
    app: wordpress-app
    tier: frontend
  type: LoadBalancer
```
